ACTIONS.HTML - Parcours requete/reponse backend

Document de suivi des flux HTTP du projet, depuis le front controller jusqu'a la reponse renvoyee au frontend, pour chaque route declaree.

Base de reference: index.php -> public/index.php -> app/bootstrap.php -> app/config/routes.php

2xx - Succes (ex: 200, 201, 204) 4xx - Erreur client (ex: 401, 403, 422, 429) 5xx - Erreur serveur (ex: 500)

Pipeline global (commun a toutes les routes backend)

1 Entree via index.php racine, qui delegue a public/index.php.

2 public/index.php sert le frontend SPA sur routes non-backend, sinon charge app/bootstrap.php.

3 Bootstrap Slim + basePath, puis pile middleware globale (execution a l'entree): ErrorMiddleware -> SessionMiddleware -> CsrfMiddleware -> RequestDataMiddleware -> CorsMiddleware -> RoutingMiddleware.

4 Matching de route dans routes.php, puis middlewares de route (Auth/Role/RateLimit selon endpoint).

5 ControllerActionInvoker execute Controller@action, recupere output/headers/status.

6 Reponse finale vers frontend: JSON via JsonResponder/Controller::respond, ou HTTP d'erreur uniforme via ErrorMiddleware (422 validation, 500 interne).

Modele de sortie vers le frontend

Succes payload JSON standardise (status, message, data) pour la majorite des endpoints API/metier.

Erreur metier ValidationException -> 422 avec details errors[] dans la reponse JSON.

Erreur technique exception non capturee -> 500 JSON; details internes exposes seulement si debug active.

CORS/Preflight OPTIONS -> 204 avec en-tetes CORS, traite par CorsMiddleware + route wildcard OPTIONS.

Schema visuel par etape (vulgarise)

1. Frontend envoie la requete Ex: GET /api/pays
2. Porte d'entree du backend index.php -> public/index.php
3. Filtre de securite et normalisation middlewares globaux dans bootstrap.php
4. Aiguillage vers la bonne route RoutingMiddleware + routes.php
5. Controleur execute Controller@action
6. Reponse renvoyee au frontend JSON succes / erreur 4xx-5xx

Image mentale simple: une requete passe plusieurs postes de controle, puis arrive au bon guichet, et repart avec un ticket de reponse.

Exemple complet de requete reussie

Scenario: connexion utilisateur avec POST /login.

1. Angular envoie email + mot de passe.

2. La requete entre par index.php puis public/index.php.

3. Passage dans les middlewares globaux (session, CSRF, normalisation body, CORS, routing).

4. Middleware de route LoginRateLimitMiddleware verifie qu'il n'y a pas trop de tentatives.

5. AuthController@login valide les identifiants et cree/met a jour la session.

6. Le backend renvoie une reponse JSON de succes au frontend (utilisateur connecte).

Exemple complet de requete en echec

Scenario: creation de ressource avec donnees invalides.

1. Le frontend envoie un POST avec champs manquants.

2. Le flux entre dans le pipeline global puis atteint le controleur cible.

3. La validation metier echoue et leve une ValidationException.

4. ErrorMiddleware intercepte l'erreur au niveau global.

5. Reponse uniforme envoyee au frontend: statut 422 + tableau d'erreurs detaille.

6. Le frontend peut afficher des messages champ par champ sans parser une erreur technique.

Qui fait quoi dans les middlewares

Middleware Role principal Effet concret sur la requete/reponse
ErrorMiddleware Filet de securite global Attrape les exceptions non gerees et renvoie un JSON coherent (422 ou 500).
SessionMiddleware Gestion de session Demarre/garantit la session HTTP, indispensable pour auth et CSRF.
CsrfMiddleware Protection anti-CSRF Verifie le token pour les requetes mutantes et bloque si token absent/invalide.
RequestDataMiddleware Normalisation des donnees Uniformise JSON/form-data en parsedBody pour simplifier les controleurs.
CorsMiddleware Autorisation cross-origin Ajoute les en-tetes CORS et gere les preflight OPTIONS.
RoutingMiddleware Selection de route Associe URL + methode HTTP au handler Controller@action.
AuthMiddleware Controle de connexion Refuse les requetes sans utilisateur connecte (401).
RoleMiddleware Controle des droits Autorise/refuse selon role (admin/artisan/client), sinon 403.
LoginRateLimitMiddleware Anti brute-force login Limite le nombre de tentatives de connexion et peut renvoyer 429.
