🏗️ 1. Architecture Générale

Le site utilise le framework Slim Framework 4, un micro-framework PHP moderne basé sur les PSR (PHP Standards Recommendations).

Structure du Projet

Fichier d'entrée:
index.php
Initialise l'application Slim, enregistre toutes les routes et lance l'application.
Bootstrap:
app/bootstrap.php
Enregistre l'autoloader PSR-4 personnalisé pour charger les classes de l'application.
Contrôleurs:
app/Controllers/
  • HomeController.php - Gère la page d'accueil
  • ProductController.php - Gère les produits et panier
  • ContactController.php - Gère la page de contact
Modèles:
app/Models/
  • Product.php - Représente un produit
  • Cart.php - Gère la session du panier
Vues:
app/Views/
  • layout.php - Modèle HTML global
  • home.php - Contenu de l'accueil
  • products.php - Contenu des produits
  • contact.php - Contenu du contact

Flux Général Appliqué à Tous les Contrôleurs

1. Requête HTTP
GET/POST vers une URL du site
2. Routeur Slim
index.php compare l'URL avec les routes enregistrées
3. Instanciation du Contrôleur
Création d'un objet du contrôleur approprié
4. Appel de la Méthode
Exécution de la méthode du contrôleur
5. Gestion de la Vue
Rendu HTML via Output Buffering (ob_*)
6. Réponse HTTP
Envoi du HTML au navigateur avec les headers appropriés

⭐ 2. Action 1: Afficher la Page d'Accueil

Accueil - Page Principale
GET
GET /

Cheminement Détaillé

1. Requête du Navigateur
GET http://localhost/
2. Routage (index.php, ligne 35-37)
$app->get('/', function (Request $request, Response $response) { ... })
3. Instanciation du Contrôleur
$controller = new HomeController();
4. Appel de la Méthode
$controller->index($request, $response)
5. Dans HomeController::index()
• Initialise $pageTitle = 'Accueil - Épicerie BIO NATURE'
• Commence le buffering: ob_start()
• Inclut app/Views/home.php
• Récupère le contenu: $mainContent = ob_get_clean()
6. Rendu du Layout
• Nouveau buffering: ob_start()
• Inclut app/Views/layout.php
• Le layout utilise $mainContent et $pageTitle
• Récupère le HTML final: $output = ob_get_clean()
7. Envoi de la Réponse
$response->getBody()->write($output);
return $response->withHeader('Content-Type', 'text/html; charset=utf-8');

Résumé de l'Action

Affiche la page d'accueil du site avec informations sur l'épicerie
Pas d'interaction avec la base de données ou session
Rendu simple et direct de la vue HTML

➕ 4. Action 3: Ajouter un Produit au Panier

Ajouter au Panier
POST
POST /products/add/{id}

Exemple: POST /products/add/1 (ajouter le produit avec l'ID 1)

Cheminement Détaillé

1. Requête du Navigateur
Formulaire HTML avec POST
form method="POST" action="/products/add/1"
2. Routage (index.php, ligne 44-47)
$app->post('/products/add/{id}', function (Request $request, Response $response, array $args) { ... })
3. Récupération de l'ID
$args['id'] est extrait de l'URL
Exemple: /products/add/3 → $args['id'] = '3'
4. Instanciation du Contrôleur
$controller = new ProductController();
5. Appel de la Méthode
$controller->addToCart($request, $response, $args)
6. Dans ProductController::addToCart()
• $productId = (int)$args['id'];
• $cart = new Cart();
• $cart->addProduct($productId);
(Appelle le modèle Cart)
7. Dans Cart::addProduct()
• $product = Product::getProductById($productId);
• Vérifie que le produit existe
• Ajoute ou augmente la quantité dans $_SESSION['grocery_cart']
📦 Stockage en Session:
$_SESSION['grocery_cart'][1] = ['product' => Product Object, 'quantity' => 1];
8. Redirection
return $response->withStatus(302)->withHeader('Location', '/products');
(Redirection HTTP 302 vers la page produits)

Résumé de l'Action

Ajoute ou augmente la quantité d'un produit au panier
Utilise la session PHP pour persister les données du panier
Valide l'existence du produit avant l'ajout
Redirige l'utilisateur vers la page produits après l'ajout
Le panier persiste entre les pages grâce à la session

➖ 5. Action 4: Retirer un Produit du Panier

Retirer du Panier
POST
POST /products/remove/{id}

Exemple: POST /products/remove/1 (retirer le produit avec l'ID 1)

Cheminement Détaillé

1. Requête du Navigateur
Formulaire HTML avec POST
form method="POST" action="/products/remove/1"
2. Routage (index.php, ligne 49-52)
$app->post('/products/remove/{id}', function (Request $request, Response $response, array $args) { ... })
3. Récupération de l'ID
$args['id'] contient l'ID du produit à retirer
Exemple: /products/remove/3 → $args['id'] = '3'
4. Instanciation du Contrôleur
$controller = new ProductController();
5. Appel de la Méthode
$controller->removeFromCart($request, $response, $args)
6. Dans ProductController::removeFromCart()
• $productId = (int)$args['id'];
• $cart = new Cart();
• $cart->removeProduct($productId);
(Appelle le modèle Cart)
7. Dans Cart::removeProduct()
unset($_SESSION['grocery_cart'][$productId]);
(Supprime complètement le produit du panier)
📦 Effet sur la Session:
Le produit avec l'ID spécifié est supprimé du panier.
S'il y a plusieurs quantités, tout est supprimé (pas de gestion partielle).
8. Redirection
return $response->withStatus(302)->withHeader('Location', '/products');
(Redirection HTTP 302 vers la page produits)

Résumé de l'Action

Supprime complètement un produit du panier
Utilise unset() pour supprimer l'entrée de la session
Ne supprime que la quantité, le produit disparaît du panier
Redirige vers la page produits
Similaire à addToCart mais avec opération inverse

🗑️ 6. Action 5: Vider le Panier Complètement

Vider le Panier
POST
POST /clear-cart

Cheminement Détaillé

1. Requête du Navigateur
Bouton ou lien dans le formulaire HTML
form method="POST" action="/clear-cart"
2. Routage (index.php, ligne 54-58)
$app->post('/clear-cart', function (Request $request, Response $response) { ... })
3. Instanciation du Modèle
$cart = new Cart();
(Pas de contrôleur, ligne de route directe)
4. Appel de la Méthode
$cart->clearCart();
5. Dans Cart::clearCart()
$_SESSION['grocery_cart'] = [];
(Établis le panier à un tableau vide)
📦 Effet sur la Session:
Tous les produits sont supprimés du panier.
La clé 'grocery_cart' existe toujours mais contient un tableau vide.
6. Redirection
return $response->withStatus(302)->withHeader('Location', '/products');
(Redirection HTTP 302 vers la page produits)

Résumé de l'Action

Vide complètement le panier en une seule action
Route directe sans contrôleur ou vue
Réinitialise la session du panier
Redirige vers la page produits
Plus rapide que retirer chaque produit individuellement

Flux Général d'une Requête HTTP

┌─────────────────────────┐ │ Requête HTTP │ │ (GET/POST + URL) │ └────────────┬────────────┘ │ ▼ ┌─────────────────────────┐ │ index.php │ │ (Entrée de l'app) │ └────────────┬────────────┘ │ ▼ ┌─────────────────────────┐ │ Routeur Slim │ │ (Matching URL) │ └────────────┬────────────┘ │ ▼ ┌─────────────────────────┐ │ Callback/Contrôleur │ │ (Traitement métier) │ └────────────┬────────────┘ │ ▼ ┌─────────────────────────┐ │ Vue (if needed) │ │ (Output HTML) │ └────────────┬────────────┘ │ ▼ ┌─────────────────────────┐ │ Response Object │ │ (Avec headers) │ └────────────┬────────────┘ │ ▼ ┌─────────────────────────┐ │ Réponse HTTP │ │ (HTML au navigateur) │ └─────────────────────────┘

Technologies Utilisées

Framework: Slim Framework 4 (PSR-7, PSR-15)
Langage: PHP 7.4+
Architecture: MVC (Model-View-Controller)
Pattern de Routage: FastRoute
Gestion de Session: Native PHP ($_SESSION)
Base de Données: Aucune (données en mémoire pour produits)

Gestion de la Session

La session est démarrée automatiquement dans le constructeur de Cart
Les données du panier sont persistées dans $_SESSION['grocery_cart']
La session existe tant que le cookie de session existe
Le panier persiste entre les pages grâce à la session
À la fermeture du navigateur, la session peut être perdue (paramètres PHP)
